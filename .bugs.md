# Bug worklist (from rules-vs-code review, 2026-07-12)

Compared against Mark III v2.05 rulebook + card text in material.inc.php.
Reported-bugs list from BGA studio not yet cross-referenced (needs login).

## Rule bugs ‚Äî high impact

- [ ] **Event-draw check uses actions remaining, not actions taken** ‚Äî `pass()` burglebros.game.php:4326-4327. `actions_remaining >= 2` only equals "took ‚â§2 actions" when turn had exactly 4. Wrong with Rollerskates (6), Mirror (3), Heads Up (5), Invisible Suit (5), Stamp variant same flaw. Fix: track actions granted at turn start, compare actions used.
- [ ] **Gymnastics floor connection ‚Äî rules-edition conflict, decide** ‚Äî `walkwayIsAdjacent()` burglebros.game.php:1089-1105. Code: walkway ‚Üî tile *above*, both directions ‚Äî matches 2016 FAQ ruling ("treated as the Stairs tile for that floor", zenxacred). Mark III v2.05 rulebook p16 says instead: move into Walkway from tile *below* it. Code follows old ruling; rulebook is newer. Probably leave as-is or match rulebook deliberately.
- [ ] **Walkway-down doesn't ignore entry effects** ‚Äî rulebook p22: moving downstairs from revealed Walkway ignores entering effects (Deadbolt cost, Fingerprint alarm...). Currently goes through full `handleTileMovement()` (:2047).
- [ ] **Walkway fall costs Stealth if guard in landing tile** ‚Äî burglebros.game.php:2148 calls `handlePlayerEnteredGuardSight` after fall. Rulebook p21: fall doesn't count as entering, no stealth loss.
- [ ] **Crash event missing "if no Alarm is currently active" gate** ‚Äî burglebros.game.php:2408-2412 (Change of Plans has the check; Crash doesn't).
- [ ] **Throw Voice missing "if no Alarm is currently active" gate** ‚Äî burglebros.game.php:2934-2949.
- [ ] **Switch Signs missing alarm clause + arrival effects** ‚Äî burglebros.game.php:2512-2531. Rulebook p16: if alarms active ‚Üí guard to closest alarm, deactivate, redraw destination. Also no stealth loss / camera check / alarm deactivation on guard's new tile (TODO comment admits it).
- [ ] **Freight Elevator top-floor escape not implemented** ‚Äî burglebros.game.php:2420-2433. Rulebook p15: on top floor, immediately escape if team holds all loot.
- [ ] **Operator precedence in `triggerAlarm` suppression** ‚Äî burglebros.game.php:2232. `a || b || emp && (!skip && hacker)` ‚Äî EMP alone never suppresses; hacker only applies when EMP active. Reachable: Dynamite (:2785), Juicer alarms during active E.M.P.
- [ ] **Motion exit check wrong** ‚Äî burglebros.game.php:2169: `$motion_entered && $motion_bit` should be bitwise `&`. Also `motionTileEntered` is global per turn, not per player (Rook-moved player pollutes it, :2122).

## Rule bugs ‚Äî edge cases

- [ ] **Peterman Drill + Stethoscope** ‚Äî `applyDieRoll()` (:1708-1711) recomputes safe tile as *player's* tile ‚Üí wrong floor for cracked/open tokens. Also `performSafeDiceRoll` `nextState('cardChoice')` (:1689) from cardChoice state has no such transition (states.inc.php:132) ‚Üí server exception.
- [ ] **`nextPatrol` force + 2+ equally-close alarms** ‚Äî burglebros.game.php:882-903: die lands on freshly drawn patrol card instead of an alarm. Reachable when guard first revealed on a floor that already has multiple alarms.
- [ ] **Multi-alarm choice lost in loops** ‚Äî Shift Change overwrites `$choose_alarm` per floor (:4710); Shoplifting overwrites `$special_choice` per lab (:2509). Only last survives; stale `specialChoiceArg` bleeds into next guard move (:1176).
- [ ] **Heads Up escaped-player skip loop doesn't advance** ‚Äî burglebros.game.php:2447-2453: `getPlayerAfterCustom()` returns same player each iteration.
- [ ] **Dead Drop can force both Gold Bars onto one player** ‚Äî burglebros.game.php:2413-2419, no gold-bar check (trades and takeCards have one).
- [ ] **Hacker moved by Rook still triggers Fingerprint** ‚Äî `hackerDoesNotTrigger()` (:1965-1983) keys off current player, checks hacker position before move. Jammer is non-optional.
- [ ] **Keycard + Drill interpretation** ‚Äî :1660-1668 lets advanced Peterman crack remotely if he holds Keycard; RAW says holder must be in Safe tile. Decide/document house ruling.
